## 教授提問與討論

### 問題 1：初次確診個案的排除策略

**背景**：
- 目前資料集約 99% 初次健檢是健康的個案
- 僅有少數初次健檢已有異常

**問題**：
1. 要不要排除初次健檢已確診的個案？
2. 使用滑動窗口時，要不要排除窗口起點已有疾病的個案？

**相關文獻**：
- Kanegae et al. (2020) 排除已有高血壓診斷者，只預測「新發」高血壓

**建議實驗設計**：

| 組別 | 說明 |
|------|------|
| A（目前做法） | 包含所有個案（含初次已確診） |
| B | 排除初次健檢已確診的個案 |
| C | 滑動窗口中，排除窗口起點已有疾病的樣本 |

**評估指標**：比較三組的 AUC

**狀態**：⏳ 待實驗

### 問題 2：增加傳統統計學方法的比較

**背景**：
- 目前已有 LR（Logistic Regression）作為傳統方法的代表
- 教授建議增加更多傳統統計方法進行比較

**可考慮的傳統統計方法**：

| 方法 | 說明 | 目前狀態 |
|------|------|----------|
| Logistic Regression | 二元分類標準方法 | ✅ 已有 |
| Naive Bayes | 基於貝氏定理的分類器 | ⏳ 待加入 |
| Linear Discriminant Analysis (LDA) | 線性判別分析 | ⏳ 待加入 |
| Cox Proportional Hazards | 存活分析方法 | ⏳ 待評估 |
| Framingham Risk Score | 傳統心血管風險評分 | ⏳ 待評估 |

**討論**：
- Naive Bayes 和 LDA 實作簡單，可快速加入比較
- Cox Model 需要時間到事件的資料，可能不適用
- Framingham 是針對心血管疾病，非直接針對三高

**狀態**：⏳ 待討論與實驗

### 問題 4：模型比較表格加入「類型」欄位

**背景**：
- 目前模型比較表格只列出模型名稱與 AUC
- 教授建議加入「類型」欄位，區分不同方法的類別

**分類方式**：

| 類型 | 模型 |
|------|------|
| 傳統統計學 | Logistic Regression, Naive Bayes, LDA |
| 機器學習 | Decision Tree, Random Forest, XGBoost, SVM, MLP |
| 符號迴歸 | PySR |

**效益**：
- 強化「傳統統計學 vs 機器學習」的比較框架
- 讓讀者更容易理解各方法的定位
- 論文貢獻更清晰：不只比較模型，也比較方法論類型

**狀態**：⏳ 待更新表格
