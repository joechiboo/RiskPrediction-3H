## Slide 17：類別不平衡處理

### 🟡 改動類型：加 framing 句（呼應 Slide 11）

### 現狀問題

目前內容是「5 方法對比 + AUC 差異 < 0.2% / Sens 提升 50-70 個百分點 / class_weight 最簡單有效」 — 數據清楚，但**缺少對「為什麼這個提升重要」的 framing**。

### 為什麼要改

Slide 11 我們鋪了一個梗：
> 「LR 的穩定性是後面 Slide 17 用 `class_weight` 拉高敏感度的前提」

Slide 17 應該**收這個梗**，把「class_weight = 技術選擇」改寫成「**class_weight = 篩檢需求驅動**」。

### 建議加什麼

**在標題下方加 framing 句**：

> **篩檢場景的失敗模式是漏抓 — Sensitivity 是命脈。但 default 設定下 sens 只有 0.041（高血壓），等於形同虛設。本頁的核心：用 `class_weight='balanced'` 把 sens 拉到 0.698，AUC 完全不變 — 用零成本換 17 倍 sens。**

**重點 punch line**（取代或補充現有 3 個 bullet）：

- ✓ **5 種方法 AUC 差異 < 0.2%** → 模型選對了，bias 處理方法不是 bottleneck
- ✓ **Sens 提升 50-70 個百分點**（高血壓 0.041 → 0.698 = **17 倍**）
- ✓ `class_weight='balanced'` 最簡單有效 — **零額外實作成本**
- → 對篩檢場景：**用零成本換 17 倍漏抓改善，這就是 LR + class_weight 的價值**

### 講稿建議（60 秒）

```
這頁回答 Slide 11 留下的伏筆：穩定性是前提，敏感度是命脈。
篩檢工具最害怕的失敗模式是漏抓真病人，sens 就是這個能力的指標。

[指表] 但 default 設定下，高血壓 sens 只有 0.041，等於 1000 個真
病人只抓到 41 個。我們比較了 5 種類別不平衡處理方法。

關鍵發現有兩個。第一，所有方法的 AUC 差異 < 0.2%——意思是模型的
鑑別能力幾乎不變，所有方法都是在「閾值/權重」層面動手腳，不是
在「能力」層面。

第二，class_weight='balanced' 把 sens 從 0.041 拉到 0.698 — 提
升 17 倍，且 AUC 完全不變。SMOTE、ADASYN、欠採樣也都拉得到類似
程度，但 class_weight 是零實作成本——只是在訓練時改一個參數。

對篩檢場景：用零成本換 17 倍漏抓改善——這就是 LR + class_weight 
為何是本研究推薦的部署組合。
```

### 預備 Q&A

| 問 | 答 |
| --- | --- |
| 為什麼 5 種方法 AUC 一樣？ | AUC 是閾值無關的指標，反映「對所有可能閾值的排序能力」。class_weight 等方法本質上只是調整決策閾值的隱式效果，沒改變模型的「排序能力」 |
| 為什麼選 class_weight 不選 SMOTE？ | 零實作成本（只改一個參數）+ 效果一樣好。SMOTE 需要合成新樣本，邊界決策上可能引入 artificial pattern |
| sens 0.698 在臨床上夠嗎？ | 取決於後續確診流程的成本。若篩檢後可低成本確認（如再抽一次血），0.698 可接受；若確認成本高，可進一步調權重提升 sens（會犧牲 spec）|

## Slide 18：MTL vs STL（多任務 vs 單任務）

### 🔴 改動類型：加「為何 MTL 沒贏」三原因 + 效率對比

### 現狀問題

目前 Slide 18 內容：
- ✓ 完整數據表（3 疾病 × 3 指標 × MTL/STL）
- ✓ 一句「推測原因：三項疾病的主要特徵差異大」

**問題**：只用一句話帶過原因，**口委大概率追問「具體哪三個原因？」**。論文 Ch7 §6 已有完整三原因，沒用上太可惜。

### 為什麼要改

口試的「為什麼 MTL 沒贏」是經典難題：
- 結果負面（MTL 沒比 STL 好），需要強解釋
- 不解釋好會被當成「方法錯了」
- 解釋好反而變成「方法對了、發現有價值」

論文 Ch7 §6 已寫好三原因，**直接搬上來**。

### 建議加什麼

**重排版面**：把「為何沒贏」獨立成一個 block，三個原因 explicit。

#### 為什麼 MTL 在這個場景沒贏？三原因

| # | 原因 | 證據 |
| :---: | --- | --- |
| ① | **任務相關性偏低** | 三疾病標籤相關性弱（Phi 係數 < 0.1）→ 共享特徵表示無法同時服務三項任務 |
| ② | **資料規模充足** | 13,514 筆紀錄已足夠 STL 獨立訓練 → MTL 的「資料增強」效益不顯著 |
| ③ | **任務難度不均** | 高血糖 0.93 vs 高血壓 0.73 → 共享層的梯度被較易任務主導 |

#### MTL 的小確幸：效率優勢

- **參數量 -66%**（共享參數減少）
- **訓練時間 -31%**
- → 若部署資源受限，MTL 仍可考慮（**用 0.9% AUC 換 66% 參數**）

### 講稿建議（80 秒）

```
這頁回答一個經典問題：同時預測三高 vs 個別預測，哪個更有效？

[指表] 結果：MTL 在 AUC 上略低 0.1-0.9%，Sens / Spec 差異也都在
1.7% 以內——可以說 MTL 沒有明顯優勢，但也沒明顯劣勢。

[指三原因 block] 為什麼 MTL 沒贏？三個原因：

第一，任務相關性偏低。三疾病的標籤相關性 Phi 係數 < 0.1——意味
著「同一個病人三高同時陽性」的傾向不強，共享特徵表示無法同時
有效服務三項預測。

第二，資料規模充足。13,514 筆紀錄對單一任務已經很充足，STL 不
缺資料，所以 MTL 的「跨任務資料增強」效益不顯著。

第三，任務難度不均。高血糖 AUC 高達 0.93，高血壓只有 0.73。共
享層在訓練時，梯度容易被「比較好做的高血糖任務」主導，反而拖
累比較難的高血壓。

不過 MTL 有一個小確幸：參數量減少 66%、訓練時間減少 31%。對部
署資源受限的場景（邊緣裝置、即時推論），仍是合理選擇——這也是
我們做這個對比的價值：**負面結果本身就是一個有用的發現**。
```

### 預備 Q&A

| 問 | 答 |
| --- | --- |
| MTL 沒贏，那做這個實驗的意義是什麼？ | 三點：① 印證 Tsai et al. (2024) 在台灣健保資料的類似觀察 ② 量化效率優勢（-66% 參數、-31% 時間）③ 提供未來研究方向參考（任務相關性低時 MTL 不適用）|
| 三疾病不是常共病嗎？為什麼相關性低？ | 共病 ≠ 同時發病。在縱向預測場景下，同一年「同時被診斷出三高」的個體相對少。Phi 係數量化的是「同時陽性 vs 同時陰性 vs 各種組合」的關聯強度，不是純粹的疾病關聯 |
| 改用其他 MTL 架構（如 cross-stitch、PCGrad）會贏嗎？ | 可能，但本研究選擇 hard parameter sharing 是因為其結構最簡單、可解釋。複雜架構即使贏也牽涉「贏在架構還是調參」的混淆，留為 future work |

## Slide 21：研究貢獻與限制

### 🟡 改動類型：加 外部驗證細節 + 美國合成資料 negative finding

### 現狀問題

目前限制段：
- 缺乏生活型態資料
- 無外部驗證資料集
- 未做校準度分析
- 用藥資訊缺失

**問題**：
1. 「無外部驗證」太單薄，沒解釋「我們試過了但失敗」
2. **美國合成資料 negative finding 沒寫進去** — 浪費了一個重要的學術觀察

### 為什麼要改

口委大概率問「**你有沒有嘗試外部驗證？**」
- 答「沒有」= 弱
- 答「有試過合成資料但失敗了，這個 negative result 反而是個發現」= **強**

把「失敗的嘗試」變成「研究觀察」是學術寫作的高招。

### 建議改寫限制段

**舊版**：

```
- 缺乏生活型態資料（飲食、運動、吸菸）
- 無外部驗證資料集（外部效度受限）
- 未做校準度分析（calibration plot / Hosmer-Lemeshow）→ SCI 版本優先補上
- 用藥資訊缺失 → 標籤定義不精確
```

**新版**：

```
- 缺乏生活型態資料（飲食、運動、吸菸）

- 外部驗證：本研究嘗試以美國合成資料驗證 generalization，但 
  performance 明顯下降，揭示 **模型可能捕捉了東亞族群特有的 
  pattern，且合成資料無法充分代表真實分布偏移**。這個觀察反映
  了 ML 醫療應用中「跨族群驗證」的根本挑戰，亟待真實 multi-
  ethnic cohort 驗證

- 未做校準度分析（calibration plot / Hosmer-Lemeshow）→ SCI 
  版本優先補上 + 時間切分驗證（用 2010-2014 train、2015-2018 
  test）作為外部驗證替代品

- 用藥資訊缺失 → 標籤定義不精確（特別是高血壓「控制中 vs 真陽
  性」難以區分）
```

### 講稿建議（補充段，~30 秒）

```
特別說一下外部驗證。我們其實有嘗試用美國合成資料驗證模型的跨
族群 generalization，但 performance 明顯下降——這個 negative 
finding 反而是個有意義的觀察：

模型可能捕捉了東亞族群特有的 pattern，而合成資料不足以代表真
實的分布偏移。要真正評估跨族群 generalizability，需要真實的 
multi-ethnic cohort 資料，這是後續 SCI 版本的優先工作之一。

我們也計畫用「時間切分驗證」（2010-2014 train、2015-2018 test）
作為外部驗證的折衷替代，可在現有資料集上立即實施。
```

### 預備 Q&A

| 問 | 答 |
| --- | --- |
| 為什麼用合成資料？真實資料拿不到嗎？ | 真實 multi-ethnic 健檢資料涉及跨國機構合作 + IRB，碩士論文時程內無法取得。合成資料是次優選擇，至少可初步檢視 distribution shift 影響 |
| 合成資料表現不好，怎麼確定模型不是真的「不好」？ | 兩個判讀方向：① 模型在原資料集上 AUC 穩定（CV + 過擬合分析 + 符號回歸公式收斂）→ 模型本身合理 ② 合成資料表現不好 → 反映 distribution shift 真實影響，不是模型錯 |
| 時間切分驗證為什麼可以當外部驗證替代？ | 不完全等同，但能捕捉「時代變化」這個 OOD 維度（如健康趨勢變化、量測標準演進）。文獻 (TRIPOD 2024) 也認可 temporal validation 作為外部驗證的合理替代 |
