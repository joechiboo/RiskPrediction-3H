## 2026-06-02 進度 memo（明天繼續）

### Slide 11 過擬合分析 — 已決定走「版本 B」

**設計**：上半保留 class-level 表 + 下半新增 per-disease 對照表

下半表內容：

| 疾病 | 最佳模型 | 最佳 Gap | LR Test | LR Gap | LR 落後 |
| --- | --- | :---: | :---: | :---: | :---: |
| 高血壓 | RF 0.743 | 0.25 | 0.721 | 0.01 | -0.022 |
| 高血糖 | LR 0.938 | 0.01 | 0.938 | 0.01 | — |
| 高血脂 | LR 0.867 | 0.01 | 0.867 | 0.01 | — |

結論句：**LR 用 2% 的 AUC 換 25 倍穩定性**

### 講稿草稿（50 秒版，待整合 framing 句）

```
這頁核心一句話：LR 用 2% 的 AUC 換取 25 倍的穩定性，這就是選它做主
模型的真實原因。

[指上半表]
上半表是三類模型的 train-test gap。傳統統計如 LR、NB、LDA，gap 都
不超過 0.01，幾乎無過擬合。樹模型 RF、XGB、LGBM 的 gap 是 0.06 到
0.25，過擬合明顯。神經網路 MLP 居中，gap 約 0.02。

[指下半表]
但要回答「LR 是不是最強」要分疾病看。高血壓最佳模型是 RF，0.743；
LR 是 0.721，落後 0.022。但 RF 的 gap 高達 0.25，LR 只有 0.01 —
相差 25 倍。高血糖、高血脂 LR 直接奪冠。

換句話說：LR 在最差的疾病只落後 RF 兩個百分點，卻換來 25 倍的穩
定性。這是論文選 LR 為主模型的真實邏輯。
```

### 🔴 明天要決定的事

**framing 句**（CV vs 過擬合分析的互補性）—— 開頭加哪個版本？

- 版本 X：「過擬合分析跟交叉驗證互補：**CV 看模型有多強，過擬合分析看模型有多『真』**」
- 版本 Y：「**CV 告訴我們模型考多少分，過擬合分析告訴我們模型是真懂還是死背**」

→ 這個 framing 是回應「CV 不就是測泛化？為什麼還要過擬合分析」的疑問。**口委大概率會問**。

### CV vs 過擬合分析（給自己的概念筆記，明天可融入講稿）

| 維度 | CV | 過擬合分析 |
| --- | --- | --- |
| 看的數字 | Test AUC | **Train + Test + Gap** |
| 答問 | 模型對 unseen 多強？ | 模型「記憶 training」多深？ |
| 範圍 | In-distribution 泛化 | 間接推測 OOD 脆弱度 |

**學生比喻**（簡單版可用於 Q&A）：
兩學生 Test 都 80 → A：Train 100（樹模型，部署崩到 60）；B：Train 81（LR，部署仍 78）。Gap 揭露 distribution shift 脆弱度。

### Q&A 預備（已擬，存進 Q&A 速記卡待整理）

| 問 | 答 |
| --- | --- |
| CV 已測泛化，為什麼還要過擬合分析？ | CV 只測 in-distribution，過擬合 Gap 反映 OOD 脆弱度 |
| Gap 小就一定好？ | 不一定，要配合 Test AUC（Gap 小但 Test 低 = underfit）|
| Gap 多大算有問題？ | 醫療 ML 文獻 0.05 是警戒線、> 0.1 通常會被質疑 |
