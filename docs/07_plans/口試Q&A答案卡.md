## 一、概念類

### Q1. 什麼是泛化（Generalization）？

**直接答**：模型在「沒見過的資料」上的表現能力。

**白話比喻**：學生背 100 題考古題答案——
- 考試考同樣 100 題 → 100 分（沒學會）
- 考試考新題型 → 0 分（沒泛化）
- 真正學會 = 能把舊知識套用到新案例

**ML 對應**：訓練集是讀過的書、測試集是新題、Test AUC 是泛化能力的量化指標、CV 是把資料切 5 份輪流當測試集做多次「測泛化」取平均。

---

### Q2. CV 不是預防過擬合嗎？

**直接答**：**否**。CV 是體溫計（量過擬合），不是退燒藥（防過擬合）。

| 工具 | 角色 |
|------|------|
| 交叉驗證 (CV) | **測量**模型在未見資料的表現 |
| 正則化 / 剪枝 / Early Stopping | **降低**過擬合的「治療手段」 |

**完整論述**：CV 能誠實估計 Test AUC，但模型本身的記憶傾向不會改變。RF 過擬合的話，做 CV 只是讓你「**看到**」它過擬合，不會讓它停止過擬合。要降低過擬合得改模型容量（複雜度、正則化、early stopping）。

**追問追答（如果委員繼續問「測試集 AUC 還 OK 不就好了嗎？」）**：

CV 的 test fold 跟 train fold 是同一群人（杭州同一世代），所以即使 RF train AUC 1.00、test 0.74，CV 報告的「平均 test 0.74」看起來還 OK — 但這只代表 **in-distribution 泛化**。

部署到台灣（不同 distribution）會崩，因為：
- **真正告訴你「死背 vs 真懂」的是 train-test gap**（RF 0.25 vs LR 0.01）
- **不是 CV test AUC 的絕對值本身**

**學生考試類比**（最直覺）：

| 學生 | 課本背熟度 | 期末考（同學區）| 轉學考（換考場） |
| :---: | :---: | :---: | :---: |
| **A 死背型** | 100% | 73 分（運氣好）| **50 分** |
| **B 真懂型** | 75% | 73 分 | **70 分** |

→ **CV 期末考分數相同看不出差別，gap 才看得出 A 死背、B 真懂**
→ 對應本研究：**LR Gap 0.01 = B 型 → 部署台灣較穩定；RF Gap 0.25 = A 型 → 部署易崩**

**關鍵記憶句**：「**CV 是體溫計、不是退燒藥**」「**CV 測考試分數、Gap 測死背真懂**」

## 二、方法選擇類

### Q3. SHAP 為什麼用 XGBoost 不用 LR？

**最強論述**（分工框架）：

> 本研究的設計是「**用 XGBoost+SHAP 找重要特徵，用 LR 部署**」這個分工：
>
> 1. **XGBoost+SHAP** = 特徵排序引擎，能捕捉非線性互動，挑出的 Top 特徵是經過嚴格篩選的 robust 特徵
> 2. **LR** = 部署模型，簡單、可解釋、不過擬合，特徵選擇實驗（§6.4 表 6-11）驗證用這些 Top 特徵在 LR 上也有效（Top 2 達 AUC 0.743，與全特徵幾乎一樣）
>
> 這個分工讓特徵重要性判斷不依賴單一模型，邏輯鏈閉合。

**4 個可辯護理由**：
1. SHAP 在樹模型有 TreeExplainer（精確、快速）；LR 的 SHAP 要 KernelExplainer（慢、近似）
2. LR 係數本身就可解釋，不需要 SHAP 機制
3. XGBoost 是「非線性壓力測試」——能捕捉互動的模型挑出的特徵更 robust
4. 方法論慣例（Lundberg & Lee 2017 原始論文主要 demo 樹模型）

**⚠️ 追問風險**：「XGBoost SHAP 排序 vs LR 係數排序是否一致？」→ 演練前最好實際驗證一下，準備備援。

### Q4. Logistic Regression 跟 Linear Regression 差在哪？

| | Linear Regression | **Logistic Regression** |
|---|---|---|
| 任務 | 回歸（連續值） | **分類**（0/1 或機率） |
| 輸出 | 任意實數 | **機率 (0,1)** ← sigmoid 壓縮 |
| 公式 | $y = \beta_0 + \beta_1 x_1$ | $P(y=1) = \frac{1}{1+e^{-(\beta_0+\beta_1 x_1)}}$ |
| 本研究 | ❌ | ✅ |

**「為什麼叫 Regression 卻是 Classification？」**：因為它在 **log-odds 空間**做的是線性回歸 $\log \frac{P}{1-P} = \beta_0 + \beta_1 x_1$。歷史上是先有「對 logit 做線性回歸」這個方法才得名 Logistic Regression。

**係數解讀**：
- β > 0 → 該特徵增加 → 患病機率增加
- exp(β) = **Odds Ratio**：特徵 +1 單位時患病勝算的倍數

**⚠️ 口語用詞**：講「邏輯斯迴歸」最安全；「邏輯回歸」可接受；**不可以**說「線性回歸」。

## 三、結果解釋類

### Q5. 高血壓 AUC 為什麼比血糖血脂低？

**標準答**：**不是模型不行，是標籤本身難預測**。

**根因不是量測，是生理本質**：

| 指標 | 變異本質 |
|------|---------|
| **血壓** | **瞬時訊號**——24 小時可變動 30-40 mmHg，受姿勢、情緒、活動、咖啡因、白袍效應影響 |
| **血糖（FBG）** | 經 8 小時禁食**消除餐後波動**，已標準化 |
| **血脂（TC）** | 慢變化指標（週為單位），單次測就有代表性 |

**臨床側面證據**：AHA 指引要求高血壓診斷至少 2 次門診、每次 2 個讀數；**ABPM**（24 小時動態血壓）和家用血壓儀的存在，就是因為**單次門診血壓被公認不可靠**——連標準化臨床量測都這樣，何況研究用的健檢數據。

**結論一句話**：血壓的訊號雜訊比本質就低，所有模型都受限於這個物理天花板。

### Q6. 過擬合分析怎麼做？為什麼要做？

**【做法】**（30 秒）：
> 在 5-fold StratifiedGroupKFold 架構下，每折除了測試集 AUC，**同時記錄訓練集 AUC**。5 折取平均後，並列比較 Train vs Test，計算 Gap。Gap 大 → 模型在訓練集學得好但推不到測試集 → 過擬合證據。

**【為什麼做】**（45 秒，3 個理由）：

1. **CV 本身不防過擬合，只測泛化**——很多人誤以為做了 CV 就不會過擬合，但 CV 是體溫計、不是退燒藥
2. **支持「為什麼選 LR 當主模型」**——表 6-5 顯示樹模型 Gap 0.06-0.25，傳統統計 ≤ 0.01，LR 泛化最穩定，後續主實驗用 LR 有方法學依據
3. **學術誠信**——只報告 Test AUC 不夠誠實，揭露 Gap 是負責任的報告方式

**關鍵記憶句**：「**CV 是體溫計、不是退燒藥**」

**【追問：樹模型 train 1.00 但 test 還有 0.74，會不會是實驗做錯？】**

否，這是樹模型的本性，不是 bug。三點：

1. **Test 0.74 不是「正常」，是「過擬合後碰巧同分布還能用」**——RF 能無限分裂到記住每個訓練樣本（train→1.00），但 test fold 跟 train fold 同屬杭州世代、分布相近，背的東西還用得上 → test 仍 0.74。換成台灣資料（不同分布）會掉更多。
2. **真正要看的是 Gap 0.25，不是 test 絕對值**——Gap 量化「模型多依賴記憶、換分布多脆弱」。
3. **LR 為何 gap 只有 0.01**——LR 只有一條線性邊界，沒能力死記，train/test 都被線性能力限制 → gap 小。

**反向檢查表（證明不是 data leakage / bug）**：

| 做錯的症狀 | 本研究 |
| --- | --- |
| Train < Test（標籤錯位） | ✅ Train > Test，方向正常 |
| 所有模型 Gap = 0（test 洩漏） | ✅ LR 0.01、RF 0.25，有區別 |
| Test AUC 高到離譜（1.00）| ✅ test 0.74 合理 |
| 同病人跨 train/test | ✅ 用 StratifiedGroupKFold 防了 |

**關鍵記憶句**：「**train 1.00 是樹的本性，gap 0.25 才是脆弱度**」

## 四、限制與部署類

### Q8. UA（尿酸）不是公費健檢項目，模型在基層健檢站能用嗎？

**直接破題（已查證，成健有 UA）**：

> 尿酸其實**是成人預防保健的標準檢驗項目**。以新北市 2020 成健套檢為例，項目代碼 **9013 Uric acid** 就在其中，與總膽固醇（9001）、肌酸酐（9015）、空腹血糖（9506）、eGFR（9515）等都**公費給付、不需自費**。所以基層健檢站取得 UA 沒有問題，本研究的生物標記幾乎全對應成健常規項目，這正是「低成本部署」的基礎。

**備援（若追問「萬一某地成健沒 UA」）**：
> 本研究 §6.4 特徵選擇實驗顯示 **Top 2 特徵即可達 95%+ 全特徵效能**，而 Top 2 是各疾病的診斷指標本身（SBP、FBG、TC 的兩個時間點），都是公費必含；UA 在 SHAP Top 10 但邊際貢獻 < 0.5%，精簡部署時可不依賴。

**優勢**：使用者為醫事檢驗所資訊主任，此題可用專業背景直接秒答（代碼 9013）。此題偏細，資訊背景口委（許揚、王建民）不太會問；蕭醫師婦產科亦非健檢專科——**列為冷門備用**。

### Q15. 為什麼要做效度分析？還差什麼？

**為什麼要做**：模型的「準」不只 AUC 高一個維度，至少 **5 個面向**：

| 效度面向 | 量測方法 | 本研究 |
|---------|---------|--------|
| **鑑別力** (Discrimination) | AUC-ROC | ✅ 完整做了 |
| **校準度** (Calibration) | Calibration plot, Hosmer-Lemeshow, Brier | ❌ 未做（§8 限制）|
| **外部效度** (External validity) | 跨資料集驗證 | ❌ 未做（§8 限制）|
| **臨床效用** (Clinical utility) | DCA, NRI, IDI | ❌ 未做（§8 限制）|
| **面向效度** (Face validity) | 多筆臨床專家評估 | ⚠️ 只有 n=1 個案 demo |

**還差什麼**（口試問到時主動列出）：

1. **Calibration plot** + Hosmer-Lemeshow test + Brier score
2. **外部驗證資料集**（跨健檢中心 / 地區 / 族群）
3. **Decision Curve Analysis (DCA)** —— 臨床淨效益
4. **Subgroup analysis** —— 年齡、性別、BMI 分層
5. **臨床專家評估的 face validity** —— 多筆典型案例

**口試講法**：
> 本研究目前完整做了**鑑別力**這一面向（AUC + 5-fold CV），但完整的效度分析至少還有**校準度、外部效度、臨床效用**三大塊未做，論文 §8 已明確列為 limitation。**SCI 階段優先補 calibration plot 和時間切分驗證**（替代外部驗證的折衷方案），其他列入未來工作。

## 六、評估指標類

### Q11. Sensitivity / Specificity / Precision / Recall 的關係？

| 英文 | 中文 | 公式 | 白話 |
|------|------|------|------|
| **Sensitivity = Recall** | 敏感度 | TP/(TP+FN) | 真正患病者中**抓出多少** |
| Specificity | 特異度 | TN/(TN+FP) | 真正健康者中**正確排除多少** |
| Precision (PPV) | 精確度（陽性預測值） | TP/(TP+FP) | 預測患病者中**真的患病多少** |
| AUC-ROC | 曲線下面積 | (積分) | 整體鑑別力 |

**易混點**：
- Sensitivity ≠ Precision（分母不同：真實陽性 vs 預測陽性）
- Recall = Sensitivity（同公式，只是 ML 圈 vs 醫學圈用詞）
- 你論文用 Sensitivity 是醫學慣用法

**為什麼重 AUC + Sensitivity 不重 Accuracy**：類別不平衡（陽性率 5-17%）時 Accuracy 會誤導；Sensitivity 直接量「抓得到多少病人」是篩檢核心。

### Q14. Slide 19 類別不平衡中，Sens 低會有什麼影響？

**直接答**：Sens 低 = **漏診率高**，篩檢工具最忌諱的失敗模式。

**具體數字影響**（[Slide 19](docs/04_meetings/口試演練_簡報.md) 高血壓為例）：

| 處理方式 | Sens | 1000 個真實高血壓患者中... |
|---------|-----|-----|
| 無處理 | 0.041 | 只抓出 41 個，**漏掉 959 個** |
| `class_weight='balanced'` | 0.698 | 抓出 698 個，漏掉 302 個 |

**對篩檢應用的意義**：
- **篩檢 (Screening) vs 診斷 (Diagnosis) 是不同任務**
- 篩檢工具優先 Sens：寧可後續再確認，不能漏
- 診斷工具優先 Spec：寧可漏也不能誤
- 三高預測屬篩檢 → class_weight='balanced' 把 Sens 從 4-34% 拉到 70-86%

**Trade-off 老實講**：
- Sens ↑ 通常伴隨 Spec ↓（誤警告變多）
- 但 **AUC 沒變**（差異 ≤ 0.2%）→ 整體鑑別力沒損失，只是把閾值調到偏陽性方向

### Q12. 為何評估標準選用 AUC？

**直接答**（4 個理由）：

1. **閾值無關**——AUC 反映模型在**所有可能閾值**下的整體排序能力，不受單一 cutoff 影響，**適合跨模型比較**（論文 [§4.4](docs/00_thesis/chapters/04_第四章_研究設計.md#L143)）
2. **抗類別不平衡**——三高陽性率 5-17%，**Accuracy 會誤導**（全預測為陰性也有 83-95% accuracy）；AUC 不受類別比例直接影響，仍能穩定評估
3. **跨研究可比較**——文獻幾乎都報 AUC（你的[表 2-1](docs/00_thesis/chapters/02_第二章_文獻探討.md)9 篇相關研究**全部用 AUC**），用同樣指標才能說「本研究與 Kanegae 0.881、Wang 0.889 是同量級」
4. **醫學標準**——TRIPOD 指引、AHA/ACC 風險預測模型評估**推薦 AUC 為主要指標**

**機率詮釋**（被深問時的學術級答案）：
> AUC = **隨機抽一個正樣本與一個負樣本，模型把正樣本排在前面的機率**。AUC = 0.85 意指模型在 85% 的隨機正負樣本對中能正確排序——這是純粹的「鑑別力」，與診斷閾值無關。

**判讀基準**（[§4.4](docs/00_thesis/chapters/04_第四章_研究設計.md#L145) 已列）：
- < 0.7：不佳
- 0.7-0.8：可接受
- 0.8-0.9：良好
- > 0.9：優秀

**主動承認 AUC 的限制**（避免被反問）：
- ❌ 不反映 **calibration**（預測機率是否準確對應實際發生率）
- ❌ 不反映**特定閾值下的臨床效用**
- ❌ 不反映**決策曲線分析 (DCA)** 的淨效益

**所以本研究輔以 Sensitivity / Specificity 報告**，未來 SCI 階段會補 calibration plot 和 DCA。

**⚠️ 追問風險**：「AUC 0.9 是否代表機率輸出準確？」→ **否**，AUC 高只代表**排序**準，**絕對機率值需要 calibration 才能保證**。這也是 Slide 22 demo 的 37.2% 不能當成「真實機率」的原因。

## 七、醫學基礎類

### Q13. SBP 是舒張壓還是收縮壓？

**直接答**：
- **SBP** = **S**ystolic Blood Pressure = **收縮壓**（高的那個）
- **DBP** = **D**iastolic Blood Pressure = **舒張壓**（低的那個）
- 例：120/80 mmHg → **120 = SBP**（收縮）、**80 = DBP**（舒張）

**記憶法**：
- **S**ystolic → 心臟 **S**queeze（收縮、擠壓）時的壓力 → 比較高
- **D**iastolic → 心臟 **D**ilate（舒張、放鬆）時的壓力 → 比較低

**論文裡的重要事實**：
- 高血壓主要驅動特徵是 **SBP_Y-1、SBP_Y-2（收縮壓）**，不是 DBP
- PySR 高血壓公式：`0.130 × exp(SBP_Y-2)`（收縮壓）

**⚠️ 簡報待修筆誤**：
- Slide 9 「範例：舒張壓 125→140」→ 應為**收縮壓**（舒張壓 125 在臨床上極端異常，125→140 不合理；收縮壓 125→140 才是 prehypertension → hypertension 的典型變化）
- Slide 21 「0.130 × exp(舒張壓_Y-2)」→ 應為 **SBP/收縮壓**

### Q16. 代糖跟胰島素分泌的關係？（呼應 Slide 24 DGA）

**用戶原始假設核對**：

| 你的說法 | 證據評估 |
|---------|---------|
| 代糖減少血糖上升 | ✅ **正確**——代糖不含可吸收熱量、不直接升血糖 |
| 代糖欺騙身體分泌過多胰島素 | ⚠️ **證據薄弱**——這是 cephalic phase insulin response (CPIR) 假說，**動物實驗有、人類研究不一致** |
| 對糖尿病不利 | ⚠️ **方向對但機制不對** |

**較穩的機制論述**（口試講這個版本）：

> 代糖確實不會直接升血糖，但近年研究發現它可能透過**改變腸道菌相、影響葡萄糖耐受性**對代謝產生負面影響（**Suez et al. 2014 Nature**）。**WHO 2023** 建議不要使用非營養性甜味劑控制體重；**DGA 2025-2030** 也**首度明確反對阿斯巴甜、蔗糖素**——跟本研究 Slide 24 提到的觀察一致。

**⚠️ 為什麼避免講「胰島素過度分泌」**：
- Cephalic phase 假說在人類證據弱（多為動物研究）
- 蕭老師臨床經驗強，可能精準指出「機轉是什麼？」
- 改用「**腸道菌相 + 葡萄糖耐受性**」論述，引用 Suez et al. 2014 Nature 是頂級期刊，站得住腳

**Slide 24 補強建議**：

如果你打算在演練時口頭提到代糖機制，把「**胰島素過度**」改成「**腸道菌相改變、葡萄糖耐受性下降**」，引用 Suez et al. 2014 Nature 或 WHO 2023 指引。
