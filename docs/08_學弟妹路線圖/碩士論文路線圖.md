# 碩士論文路線圖（給即將升二年級的學弟妹）

> 寫於 2026-05，我口試前兩個月。把我跑過一輪的時序、坑、可分享經驗整理成這份地圖。
> 你現在的位置：**二年級上學期開始前**，正是定題目、確定資料、預跑實驗的關鍵窗口。

---

## 全局時間軸（以你二年級為主）

```
[二上 暑假]    [二上 9-10月]   [二上 11-12月]   [寒假 1月]    [二下 2月]   [二下 3-4月]   [二下 5月]    [6-7月]        [8月]
   ↓               ↓               ↓               ↓            ↓             ↓             ↓            ↓             ↓
題目/資料定案   文獻+方法設計   實驗主體跑完     初稿定稿     Word 排版    助教審稿+修訂  提口試申請  印刷/簡報/口試  離校+SCI
```

**最重要的兩句**：

1. **一月底 = 初稿（Ch1-Ch9 內容）要全部寫完**。我自己卡到這個 deadline 才有辦法二月開始排版。
2. **二月你不該還在跑實驗**。實驗該在 **12 月底前收斂**，二月只剩很小的調整（補一張圖、補一個對照組），主力是 Word 排版。

論文是 **二年級暑假就要開始**的，不是二下才開始。慢一個月，後面每個階段都會被擠壓。

## 階段 1：題目與資料定案（二上暑假，7-8 月）

### 該做什麼

- 跟指導教授確認研究主題的 **三件事**：
  1. 預測什麼（target）
  2. 用什麼資料（dataset）
  3. 用什麼方法（baseline + 想試的東西）
- 把資料**下載下來、跑通一份最小範例**（讀檔 → 切 train/test → 跑 baseline → 印 AUC）
- 開 GitHub repo + 結構化資料夾（`docs/`、`notebooks/`、`data/`），用 git 從第一天就開始追蹤
- 把所有相關論文先抓 5-10 篇放著（不用全讀，先建索引）

### 常見坑

- **題目選太大**：例如「用 AI 預測健康」太空泛。要收斂到「用 X 資料 預測 Y target 在 Z 時間窗」。
- **資料拿不到**：先確認資料**取得方式**（公開 / 申請 / 跟誰要），不要等寫到一半才發現要倫審。
- **沒跑 baseline**：很多人花三個月看論文卻沒寫過一行 code，方向會飄。**先讓 pipeline 跑起來**，再回頭精修。
- **跟教授對話頻率太低**：暑假常見現象。最少**兩週見一次**，不然方向跑掉沒人拉。

### 我的經驗切入點（你可以問我）

- **怎麼挑 Luo et al. (2024) Dryad 資料集**：當初比較過幾個 open dataset 的取捨（樣本量、追蹤期、變數豐富度）
- **為什麼三 target（高血壓/高血糖/血脂異常）一起做**：跟單 target 比的 pros/cons
- **跟教授 onboarding 的節奏**：第一次 meeting 該帶什麼、怎麼問問題、怎麼留 meeting note

## 階段 2：文獻回顧與方法設計（二上 9-10 月）

### 該做什麼

- **建文獻索引**：建一個總表（我用 [文獻總索引.md](../07_plans/文獻總索引.md)），每篇記：作者/年份/題目/資料/方法/結論/我能引用在哪
- 跑 **Related Work 的時候同步寫 Ch2**，不要等到所有文獻讀完才寫
- 確定你的 **research gap**：明確寫出「現有研究做了 A、B，但沒做 C，我做 C」
- 開始畫**系統流程圖**（資料 → 預處理 → 特徵 → 模型 → 評估），二下你會反覆改它

### 常見坑

- **只讀不整理**：讀 50 篇但 Ch2 寫不出來，因為沒分類。**讀一篇就馬上分到一個 cluster**。
- **et al. 引用搞錯作者**：我之前用 LLM 補作者全名，**15 篇有 13 篇錯**（名字、DOI、期刊都會幻覺）。**用 Crossref API 驗證**：`https://api.crossref.org/works/{DOI}`。
- **找不到 research gap**：通常是文獻看太窄，跨領域看一下（醫學 / 統計 / 工程都有用）。
- **Reference 管理一團亂**：從第一天就用統一格式（我用 IEEE，Ch9 才不會崩潰）。建議用 Zotero 或 Mendeley。

### 我的經驗切入點

- **Reference 管理流程**：我有 [Reference_Download_Index.md](../02_literature/Reference_Download_Index.md) 可以參考結構
- **et al. 展開血淚史**：可以直接看 [memory/reference-verification.md](../../memory/reference-verification.md) 學教訓
- **怎麼分文獻 cluster**：我分了「資料集」「方法」「臺灣本地研究」「特徵工程」幾類

## 階段 3：實驗主體（二上 11-12 月）

### 該做什麼

- **正式跑完整實驗**：所有模型 × 所有 target × 所有評估指標
- 結果**全部存成 csv/notebook**，不要相信記憶。**每個數字都要能追回是哪個 notebook、哪個 commit 跑出來的**
- 開始寫 **Ch3（方法）、Ch4（資料）、Ch5（實驗設計）**，邊跑邊寫
- **12 月底 = 實驗收斂死線**：可以給教授看一份完整結果表，所有主表（AUC、特徵重要性、模型比較）數字定下來。一月開始就只寫文字，不重跑大實驗

### 常見坑

- **Data leakage**：時序資料的天坑。我這個專案是用 `StratifiedGroupKFold(groups=patient_id)`，**不能用一般 KFold** 不然會洩漏同一個人的不同時間點。
- **資料格式定不下來**：我吃過「wide format vs sliding window」雙版本問題（一個 6056 樣本、一個 13514 樣本，欄位命名都不同），改一次就要重跑所有實驗。**先把資料格式定死再開始跑**。
- **實驗不可重現**：seed 沒固定、版本沒記、超參數沒存。**每個 notebook 開頭固定 seed、結尾存 config**。
- **AUC 跨表不一致**：你在 Ch5 印一個數字，Ch6 又印一個不同的，因為是不同次跑的。**最後一輪要重跑一次「定版實驗」**，所有表格從那一版抓。

### 我的經驗切入點

- **Sliding window 設計**：T1/T2 → T3 的設計、為什麼要這樣切，可以給你看 notebook
- **PySR 符號回歸怎麼接**：當初為什麼加它、跟其他 11 個模型怎麼比較
- **Notebook 命名規則**：我用 Nb01、Nb02...，方便引用，你可以抄
- **target 編碼 1/2 → 0/1 的坑**：`(df[col] == 2).astype(int)`，這個搞錯結果會反過來

## 階段 4：初稿撰寫（寒假 1 月）

### 該做什麼

- **目標：1 月底 Ch1-Ch9 全部寫完**（Markdown 版本，內容定稿）
- 二上 11-12 月跑實驗時其實 Ch3-Ch5 已經有草稿。一月主力是把 Ch1（緒論）、Ch2（文獻）、Ch6（結果分析）、Ch7（討論）、Ch8（結論）寫完
- 寫的時候**每週給教授看一章**，不要等全寫完才給。教授會修方向，越晚改越痛
- 同步準備 **摘要（中/英）、目錄、圖表清單**
- 圖表用 **向量圖**（svg/pdf），印出來才不會糊

### 常見坑

- **邊寫邊改實驗**：實驗該在 12 月底收斂。一月還在改實驗 = 後面全部塞車。**鎖實驗、只改文字**。
- **章節風格不一**：Ch1 寫得像小說、Ch3 寫得像 code 註解。**先定 style guide**：人稱、時態、列點 vs 段落、圖表標號格式。
- **IEEE 參考文獻自己排版**：超花時間。**從第一篇就用統一格式累積**，不要等 Ch9 才整理。
- **拖到二月才開始寫**：那後面排版、修訂、提口試會全部擠在一起，必崩。一月底沒寫完 = 警報。

### 我的經驗切入點

- **章節結構模板**：可以參考 [Thesis_Structure_Plan.md](../07_plans/Thesis_Structure_Plan.md)
- **Markdown 寫論文的工作流**：怎麼一邊寫一邊管圖表、引用、章節編號
- **Ch1-Ch9 各章重點**：每章該回答什麼問題、不該寫什麼

## 階段 5：Word 排版 + 教授修訂（二下 2-4 月）

### 該做什麼

- **2 月寒假**：主力是 **Markdown → Word 排版**。實驗只剩很小調整（補一張圖、改一個對照組），不再開新實驗
- **3-4 月**：教授給回饋，反覆改內容（我自己跑了初稿 I → II → III → IV 四版）
- **4 月底**：論文初稿要到「給口委看不丟臉」的狀態 = 內容定稿、排版乾淨、參考文獻齊全

### 常見坑

- **Word 排版至少吃掉兩週**：首行縮排、書籤、圖表編號、IEEE 參考文獻格式、目錄頁碼、表跨頁——每一項都是地雷。**不要低估**。
- **Markdown vs Word 雙版本同步**：改了 Word 但 Markdown 沒同步，下一輪改又翻車。**選一個當 source of truth**（我後期是 Word 當定版，Markdown 只留歷史）。
- **教授改稿很慢**：教授可能一輪 2-3 週才回。**先丟、不要等回再寫下一章**。
- **圖表編號跨章節亂掉**：圖 3-1 變圖 4-1，全章的引用都要改。**用 Word 的交叉引用功能**，不要硬寫。

### 我的經驗切入點

- **Markdown → Word 轉檔的雷**：[論文PDF排版檢查.md](../07_plans/論文PDF排版檢查.md)、[初稿VII_首行縮排修正清單.md](../07_plans/初稿VII_首行縮排修正清單.md) 都是我踩過的
- **論文格式規範**：系上有 115.04.02 版懶人包，**邊界要按它的、不要按指導手冊**（兩者有衝突，以懶人包為準）。參考 [論文格式規範摘要.md](../07_plans/論文格式規範摘要.md)
- **初稿 I → IV 的版本控管**：每版改了什麼、怎麼追蹤

## 階段 6：提口試申請（二下 5 月）

### 該做什麼

- **5 月中**：提口試申請（系辦窗口、表單、口委名單、論文初稿摘要）
- **助教格式審**：這時候助教會審你的論文格式（摘要、目錄、頁碼、邊界），給意見要當天改完
- **同步聯絡口委**：寫信問時間、確認他們收到通知
- **借場地**：跟系辦/篤行樓辦公室申請教室

### 常見坑

- **摘要寫了論文題目**：助教會退。**摘要只寫內容、不要寫題目**（題目封面已經有了）。
- **目錄列了第一章前的東西**：目錄只列「第一章開始之後」的章節，封面/摘要/誌謝不放進目錄。
- **口委聯絡太晚**：教授常常很忙，至少**提前 1.5 個月**敲時間。我這次是 5 月敲 7 月，剛好。
- **場地沒人借得到**：熱門教室要早搶。我搶到視聽館 F408 是運氣好 + 早申請。
- **助教關係**：助教 vs 教授偶爾有張力，**訊息要過誰、表單誰簽**要先問清楚，不要踩雷。

### 我的經驗切入點

- **口試申請流程**：我可以給你看實際填過的表單範本
- **助教審稿意見**：[memory/abstract-format.md](../../memory/abstract-format.md) 有他給我的具體意見
- **跟教授敲時間的信件範本**：可以直接抄

## 階段 7：印刷 + 寄口委 + 簡報準備（6-7 月）

### 該做什麼

- **6/15 前後**：論文初稿**完全定版**（口委版，不是最終版，後面還會改）
- **6/20 前後**：膠裝印刷 ×4 本（指導+2 口委+備份），寄送 ×3 口委
- **6/23 前後**：寄出，**確認收件人地址、附件**
- **6 月底**：口試簡報定版（25 頁左右、20-25 分鐘）
- **7 月初**：練習簡報，計時、預演 Q&A

### 常見坑

- **印太早改不了**：發現錯字就尷尬。**最終版 = 印刷版 + 口試後修訂版**，分清楚。
- **印刷規格錯**：邊界、頁碼、紙質、封面顏色都有規範。**先印 1 本試樣**。
- **簡報塞太滿**：25 頁 × 1 分鐘 = 25 分鐘，剛好。**不要塞 40 頁**。
- **Q&A 沒準備**：口委會問的問題其實有套路（限制、未來工作、為什麼這個方法、跟某論文差別）。**做一份 Q&A 速記卡**。

### 我的經驗切入點

- **印刷指南**：[論文印刷指南.md](../07_plans/論文印刷指南.md) 有規格、店家、價格
- **寄送口委 SOP**：附件清單、收件地址、追蹤
- **簡報 17→25 頁的擴充思路**：詳見 meeting24 的計畫
- **Q&A 速記卡**：[口試Q&A速記卡_A4.md](../07_plans/口試Q&A速記卡_A4.md) 直接抄結構

## 階段 8：口試（7 月）

### 該做什麼

- **前一天**：場地測投影、印備用講稿、確認帶筆電/轉接頭/水
- **當天**：早到 30 分鐘，跟教授先過一次流程
- **流程**：開場 → 口委介紹 → 學生報告 20-25 分鐘 → Q&A 30-40 分鐘 → 口委討論 → 宣布結果
- **記筆記**：口委的每個意見都要寫下來，後面修訂用

### 常見坑

- **設備沒測**：HDMI/USB-C/解析度不合是經典悲劇。**至少前一天去場地測**。
- **緊張到漏內容**：講稿可以放螢幕前當小抄。**重點是穩，不是順**。
- **被質疑就慌**：口委質疑 ≠ 否定，是學術討論。**承認限制、說明後續可改進**比硬辯好。
- **沒安排茶水/簽到**：細節，但會被注意到。

### 我的經驗切入點

- **F408 場地細節**：投影、座位、插座位置
- **口委風格筆記**：王建民/蕭慶華的提問風格（口試後我會補進來給你）
- **教授偏好**：簡報節奏、用詞、圖表偏好

## 階段 9：離校 + 後續（口試後 - 8 月）

### 該做什麼

- **口試後 2 週內**：根據口委意見修訂論文最終版
- **送圖書館 / 系辦**：印刷版、PDF、電子授權
- **離校手續**：學雜費、宿舍、學生證、印鑑、各種系統登出
- **離校建議在 8/15 前完成**（助教建議的時程）
- **SCI 投稿**：論文改寫成期刊版本，可以提前佈局架構

### 常見坑

- **修訂版拖太久**：口委意見當下都記得，**兩週內改完**，不然忘記細節。
- **離校系統忘了某項**：印鑑、論文系統、線上授權，常漏一個就要再跑一趟。**列 checklist**。
- **SCI 投稿沒準備**：學位論文 ≠ 期刊論文。期刊要重寫、刪 80%、加新分析。**提前佈局 outline**。

### 我的經驗切入點

- **SCI 轉換計畫**：[SCI_論文轉換計畫.md](../07_plans/SCI_論文轉換計畫.md) 有我的初步規劃
- **離校 checklist**：我跑完之後會補上一份完整版
- **修訂版控管**：怎麼追蹤每個口委意見對應的修改

## 給學弟的最後三句話

1. **行政流程比技術花時間**。寫程式跑模型是你會的，但申請、印刷、寄送、場地、規範這些**是新東西**，要當作任務排進日程。
2. **資產化你的工作**。每個 notebook、每篇文獻、每次 meeting note，都當作未來「會用第二次」來整理。學弟（你的學弟）會謝謝你。
3. **不要單打獨鬥**。卡住就問，問教授、問同學。一個下午問來的答案，自己 google 要找一週。
