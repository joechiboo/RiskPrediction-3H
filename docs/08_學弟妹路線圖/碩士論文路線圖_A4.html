九階段速覽

階段時點內容
1. 題目/資料暑假 7-8月 ● 跟教授對齊「預測什麼/用什麼資料/用什麼方法」三件事；資料下載下來、跑通最小 pipeline；建 GitHub repo。
● 題目太大、資料拿不到才發現要倫審、只看論文沒寫 code 方向會飄、跟教授見面太少（≥ 兩週一次）。
● Luo (2024) Dryad 資料集怎麼挑、三 target 一起做的取捨、跟教授 onboarding 節奏。
2. 文獻+方法9-10月 ● 建文獻索引總表（作者/年份/資料/方法/結論/能引用在哪）；邊讀邊寫 Ch2；明確寫出 research gap。
● 只讀不整理；et al. 名字被 LLM 補錯（我 15 篇錯 13）→ 用 Crossref API 驗 api.crossref.org/works/{DOI}；Reference 拖到最後才整理。
● Reference 索引格式、et al. 展開血淚史、文獻 cluster 分法。
3. 實驗主體11-12月 ● 跑完整實驗（所有模型 × 所有 target）；結果全存 csv/notebook，每個數字能追回 commit；邊跑邊寫 Ch3-5；12 月底死線。
● Data leakage（時序資料用 StratifiedGroupKFold(groups=patient_id)，不能用一般 KFold）；資料格式定不下來反覆重跑；seed/版本沒存無法重現；AUC 跨表不一致 → 最後重跑一次「定版實驗」。
● Sliding window T1/T2→T3 設計、PySR 接法、Notebook 命名規則、target=2→1 編碼坑。
4. 初稿撰寫寒假 1月 ● 1 月底 Ch1-9 全寫完（Markdown 內容定稿）；Ch3-5 暑假已草稿，一月主攻 Ch1/2/6/7/8；每週給教授看一章；同步準備中英摘要、目錄。
● 邊寫邊改實驗、章節風格不一、IEEE 參考文獻拖到 Ch9 才弄、拖到 2 月才寫後面全部塞車。
● 章節結構模板、Markdown 寫論文工作流、Ch1-9 各章該回答什麼。
5. 排版+修訂2-4月 ● 2 月寒假：Markdown → Word 排版；3-4 月教授給回饋反覆改（我跑了初稿 I→II→III→IV 四版）；4 月底要到「給口委看不丟臉」狀態。
● Word 排版至少吃兩週（首行縮排/書籤/圖表編號/IEEE/目錄頁碼）；Markdown vs Word 雙版本不同步（選一個當 source of truth）；教授 2-3 週才回（先丟、別等）。
● Markdown→Word 轉檔的雷、系上 115.04.02 版懶人包（邊界以懶人包為準）、初稿版本控管。
6. 提口試5月 ● 5 月中提口試申請（表單、口委名單、論文摘要）；助教格式審（這時才碰到助教，意見當天改完）；聯絡口委、借場地。
● 摘要寫了論文題目會被退；目錄不能列封面/摘要/誌謝；口委敲時間要提前 1.5 個月；場地早申請；助教 vs 教授偶有張力，訊息要過誰先問清楚。
● 口試申請表單範本、助教審稿意見、跟教授敲時間的信件範本。
7. 印刷+簡報6-7月 ● 6/15 論文定版；6/20 膠裝 ×4（指導+2 口委+備份）；6/23 寄出；6 月底簡報定版（25 頁、20-25 分）；7 月初練習計時+預演 Q&A。
● 印太早改不了（最終版 = 印刷版 + 口試後修訂版，分清楚）；印刷規格錯（先印 1 本試樣）；簡報塞太滿；Q&A 沒準備（口委問題有套路）。
● 印刷指南（規格/店家/價格）、寄送 SOP、簡報擴充思路、Q&A 速記卡架構。
8. 口試7月 ● 前一天場地測投影、印備用講稿；當天早到 30 分鐘；報告 20-25 分 + Q&A 30-40 分；口委每個意見都寫下來。
● 設備沒測（HDMI/USB-C 經典悲劇）；緊張漏內容（講稿當小抄、穩比順重要）；被質疑就慌（承認限制+說後續可改進，比硬辯好）。
● F408 場地細節、口委風格筆記（口試後補）、教授簡報偏好。
9. 離校+SCI7-8月 ● 2 週內改完最終版；送圖書館+系辦+電子授權；8/15 前離校；SCI 投稿可提前佈局。
● 修訂版拖太久會忘細節；離校系統漏項要再跑一趟（列 checklist）；SCI ≠ 學位論文，要重寫刪 80%。
● SCI 轉換計畫、離校 checklist（口試後補）、口委意見追蹤表。

三句話總結

  • 行政流程比技術花時間。寫程式你會，但申請/印刷/寄送/場地是新東西，要排進日程。
  • 資產化你的工作。每個 notebook/文獻/meeting note 都當「會用第二次」來整理。
  • 不要單打獨鬥。卡住就問教授、問同學。一個下午問來的答案，自己 google 要找一週。
