# 模型白話筆記 — 口試準備用

本文件整理論文中 11 個模型的白話解釋，方便口試前快速複習。

---

## 一、傳統統計方法（線性、可解釋）

### 1. Logistic Regression (LR) — 邏輯斯迴歸

**一句話**：畫一條線把生病和健康分開，輸出「生病的機率」。

**原理**：
- 把所有特徵乘上各自的權重加起來，再用 sigmoid 函數壓到 0~1 之間變成機率
- 公式：P(生病) = 1 / (1 + e^(-加權總和))
- 例如：血壓權重 0.3、血糖權重 0.5 → 加權總和越大 → 機率越接近 1

**為什麼用它**：
- 最經典的基準模型，論文做比較時幾乎都會有
- 係數可以直接解讀：「血糖每增加 1，風險變 e^β 倍」（勝算比 Odds Ratio）
- Sun et al. 的 26 篇高血壓研究中，12 篇用 LR

**弱點**：假設特徵和風險是線性關係，無法捕捉複雜的交互作用

---

### 2. Naive Bayes (NB) — 單純貝氏分類器

**一句話**：假設每個特徵互相獨立，用貝氏定理反推「給定這些檢查數值，生病的機率多少」。

**原理**：
- 貝氏定理：P(生病|數值) = P(數值|生病) × P(生病) / P(數值)
- 「Naive」= 天真地假設每個特徵獨立（例如假設血壓和血糖無關）
- 用高斯分佈（常態分佈）估計每個特徵在生病/健康族群中的分佈

**為什麼用它**：
- 計算超快，只要算每個特徵的平均值和標準差
- 和 LR 形成「判別式 vs 生成式」的理論互補

**弱點**：獨立假設在醫療資料中幾乎不成立（血壓收縮壓和舒張壓明顯相關）

---

### 3. Linear Discriminant Analysis (LDA) — 線性判別分析

**一句話**：找一個投影方向，讓兩組人（生病 vs 健康）投影後距離最大、各自最集中。

**原理**：
- Fisher 準則：最大化「類別間距離 / 類別內分散度」
- 想像把所有人的資料投影到一條線上，調整線的方向讓兩群人分最開
- 假設兩群人的資料都是常態分佈，且分佈形狀（共變異數）相同

**為什麼用它**：
- 兼具降維和分類功能
- 考慮特徵之間的相關性（共變異數矩陣）
- 和 LR、NB 形成三種不同理論基礎的線性方法比較

**弱點**：常態分佈和等共變異數假設在實際中可能不成立

## 二、機器學習方法

### 4. K-Nearest Neighbors (KNN) — K 最近鄰

**一句話**：找你最像的 K 個人，看他們多數是生病還是健康。

**原理**：
- 不建立任何模型（惰性學習），預測時才去搜尋
- 用歐氏距離計算兩個人有多「像」
- 找到最近的 K 個鄰居（本研究 K=5），多數決投票

**白話例子**：
- 你血壓 130、血糖 6.5，找到 5 個數值最接近的人
- 其中 4 個有高血壓 → 預測你也會得高血壓

**為什麼用它**：
- 「相似的人有相似的健康風險」很符合醫學直覺
- 和其他方法形成「有模型 vs 無模型」的對比

**弱點**：
- 特徵要先標準化，不然量綱大的特徵會主導距離
- 預測速度慢（每次都要算距離）
- 高維度時效果變差（維度詛咒）

---

### 5. Decision Tree (DT) — 決策樹

**一句話**：像問診流程圖，一步步問問題把人分群。

**原理**：
- 每個節點問一個問題：「血壓 > 140？」
  - 是 → 左邊走
  - 否 → 右邊走
- 用 Gini 不純度決定「先問哪個問題最好」
  - Gini = 0 表示完全純（節點裡全是同一類）
  - 每次選能讓 Gini 下降最多的分割方式

**白話例子**：
```
血壓 > 140?
├── 是 → 血糖 > 7.0?
│   ├── 是 → 高風險
│   └── 否 → 中風險
└── 否 → 低風險
```

**為什麼用它**：
- 最直觀、最好解釋的模型
- 可以直接看到分類規則

**弱點**：容易過擬合（一棵樹容易記住訓練資料的雜訊）

---

### 6. Random Forest (RF) — 隨機森林

**一句話**：種很多棵不同的決策樹，讓它們投票決定結果。

**原理**：
- **Bagging**（Bootstrap Aggregating）：
  1. 從資料中隨機抽樣（有放回），產生多個子資料集
  2. 每個子資料集各訓練一棵決策樹
  3. 每棵樹在每次分裂時，只從隨機選的幾個特徵中挑最好的
  4. 最終投票：100 棵樹中 70 棵說生病 → 預測生病

**為什麼比單棵 DT 好**：
- 每棵樹看到不同的資料和特徵，降低了過擬合風險
- 就像問 100 個醫生的意見比問 1 個更可靠

**關鍵概念**：樹和樹之間是**平行**、**獨立**的

---

### 7. XGBoost — 極端梯度提升

**一句話**：一棵一棵樹接力，每棵專門修正前一棵做錯的部分。

**原理**：
- **Boosting**（提升）策略：
  1. 第 1 棵樹做完預測後，找出它預測錯的地方
  2. 第 2 棵樹專門學習第 1 棵的錯誤
  3. 第 3 棵樹專門學習前 2 棵還沒修好的錯誤
  4. ...依此類推，越來越精準
- 目標函數同時考慮「預測準確度」和「模型複雜度」（正則化防過擬合）

**RF vs XGBoost 最重要的差別**（口試常問）：
- RF：每棵樹**獨立**訓練，最後投票（Bagging = 平行）
- XGBoost：每棵樹**接力**修正前一棵的錯（Boosting = 串接）
- XGBoost 通常更準，因為它專門針對「錯的地方」學習

---

### 8. LightGBM — 輕量梯度提升機

**一句話**：和 XGBoost 概念一樣（Boosting），但用更聰明的策略讓速度更快。

**和 XGBoost 的差異**：
- **GOSS**（梯度單側採樣）：只保留「錯很多的樣本」全部訓練，「錯很少的」只抽樣 → 減少計算量
- **EFB**（獨佔特徵綑綁）：把互斥的稀疏特徵合併 → 減少特徵維度
- **Leaf-wise 生長**（逐葉）vs XGBoost 的 Level-wise（逐層）：
  - XGBoost：每層所有節點都長 → 比較平衡
  - LightGBM：挑增益最大的葉子長 → 更快收斂但更容易過擬合

**為什麼用它**：和 XGBoost 互補比較，看兩種策略在三高預測上差異

---

### 9. Support Vector Machine (SVM) — 支援向量機

**一句話**：找一個間隔最大的分界面；如果線性分不開，用核函數把資料「升維」再切。

**原理**：
- 找一個超平面，讓兩類樣本離超平面**最遠**（最大間隔 = 最大 margin）
- 只有靠近分界面的樣本（支援向量）真正影響決策
- **核技巧**（Kernel Trick）：
  - 如果在 2D 分不開，映射到 3D 可能就分得開
  - RBF 核函數：K(x,y) = exp(-γ||x-y||²)
  - γ 越大 → 決策邊界越複雜（越彎曲）

**白話比喻**：
- 想像兩群點在桌上混在一起，你用一張紙（超平面）把它們分開
- 如果平面分不開，就把它們「拋到空中」（升維），在空中用一張紙就切得開了

**為什麼用它**：
- 理論基礎最扎實的方法之一
- 適合中小型資料集（本研究 6,056 人）
- 代表「核方法」這一類，和樹模型、神經網路形成完整比較

---

### 10. Multi-Layer Perceptron (MLP) — 多層感知器

**一句話**：多層神經元，每層做「加權求和 → 非線性轉換」，可以學到很複雜的關係。

**原理**：
- 結構：輸入層（26 個特徵）→ 隱藏層 1（64 神經元）→ 隱藏層 2（64 神經元）→ 輸出（0 或 1）
- 每個神經元做的事：
  1. 把上一層的輸出乘上權重加起來（z = Wx + b）
  2. 過一個激活函數（ReLU = max(0, z)）
- **前向傳播**：資料從左到右一層一層算過去
- **反向傳播**：算完之後，用鏈式法則從右到左算每個權重該怎麼調

**為什麼用 ReLU 不用 sigmoid？**（口試可能問）
- sigmoid 在值很大或很小時梯度趨近 0（梯度消失）→ 深層網路學不動
- ReLU 在正值區域梯度恆為 1 → 梯度可以順利傳回去

**為什麼用它**：代表神經網路方法，理論上能學到最複雜的非線性關係

**弱點**：黑箱，無法解釋為什麼做出某個預測

## 三、符號回歸

### 11. PySR — 符號回歸

**一句話**：用演化算法自動搜尋數學公式，找出最簡潔又準確的預測公式。

**原理**：
- 遺傳規劃（Genetic Programming）：
  1. 隨機生成一堆數學公式（像 `FBG × 0.3 + BMI²`）
  2. 評估每個公式的預測準確度
  3. 保留好的公式，淘汰差的
  4. 對好的公式做「交叉」（交換子公式）和「突變」（隨機改一項）
  5. 重複幾百代 → 公式越來越好
- 同時考慮「準確度」和「複雜度」→ Pareto 前沿：列出從最簡單到最複雜的最佳公式

**為什麼用它**：
- 其他 10 個模型都是黑箱或半黑箱
- PySR 產出的公式人看得懂，醫生可以直接用
- 例如：`risk = 0.3 × FBG + 0.1 × BMI - 2.5` → 不需要電腦就能算

## 四、口試高頻問題整理

### Q1：RF vs XGBoost 差在哪？
- RF = Bagging（平行、獨立、投票）
- XGBoost = Boosting（串接、接力修正錯誤）
- XGBoost 通常更準，因為專門針對錯的地方學

### Q2：為什麼要用這麼多模型？
- 涵蓋四大類方法：線性統計、樹模型、核方法、神經網路 + 符號回歸
- 不同方法有不同假設和特性，系統性比較才能知道哪個最適合三高預測
- 這也是本研究的學術貢獻之一

### Q3：SVM 的核函數在幹嘛？
- 把線性分不開的資料映射到更高維的空間
- 在高維空間中就可以用超平面切開
- RBF 核是最常用的，γ 控制決策邊界的複雜度

### Q4：為什麼用 PySR 不用 gplearn？
- gplearn 不支援 class_weight，無法處理類別不平衡
- PySR 基於 Julia，搜尋效率更高
- PySR 支援 sample_weight

### Q5：MLP 的 ReLU vs sigmoid？
- sigmoid 有梯度消失問題（值大或小時梯度 ≈ 0）
- ReLU 正值區梯度恆為 1，梯度傳播順暢
- 現在深度學習幾乎都用 ReLU 或其變體

### Q6：class_weight='balanced' 在做什麼？
- 自動把少數類的權重調高：w = 總樣本數 / (類別數 × 該類樣本數)
- 例如高血糖只有 5.5%，權重就會被放大約 18 倍
- 讓模型更重視少數類（患病者）的正確分類

### Q7：Bagging vs Boosting 怎麼記？
- **Bagging**（裝袋）：每個人獨立做，最後投票 → 降低 variance
- **Boosting**（提升）：接力修正，一個接一個 → 降低 bias
- RF 用 Bagging，XGBoost/LightGBM 用 Boosting

### Q8：LightGBM 的 Leaf-wise vs Level-wise？
- Level-wise（XGBoost）：每層全部節點都分裂 → 比較平衡不容易過擬合
- Leaf-wise（LightGBM）：每次只分裂增益最大的葉子 → 更快但更容易過擬合
- LightGBM 需要搭配 num_leaves 限制來防止過擬合
