# 為何用 T3 而非 T2 作為預測目標？

> **建立日期**：2025-12-04
> **來源**：Meeting 17 討論

---

## 研究設計

```
T1 ──────→ T2 ──────→ T3
  (間隔約2年)   (間隔約2年)

輸入特徵：X_T1, X_T2, Δ(T2-T1)
預測目標：Y_T3（是否罹患三高）
```

---

## 為什麼不用 T2？

如果用 T2 作為目標：

```
輸入：X_T1
預測：Y_T2
```

### 問題

1. **無法使用 Δ 特徵**
   - Δ = T2 - T1
   - 如果 Y 也在 T2，就是用「結果」預測「結果」

2. **資料洩漏（Data Leakage）**
   - T2 的健檢數據和 T2 的疾病狀態是**同一次檢查**得到的
   - 模型會「偷看答案」

---

## 用 T3 的好處

| 優點 | 說明 |
|------|------|
| **Δ 特徵可用** | T2-T1 的變化量可以作為輸入，預測未來（T3） |
| **真正的預測** | 用「過去」預測「未來」，有實際預警價值 |
| **時序正確** | 輸入 → 輸出 有明確的時間先後關係 |
| **臨床意義** | 給醫生約 2 年的介入時間窗口 |

---

## 簡單比喻

```
❌ 用 T2：看著今天的體重，預測今天會不會胖 → 沒意義
✅ 用 T3：看著過去兩年的體重變化，預測兩年後會不會胖 → 有預警價值
```

---

## 口述說明

> 我們選擇 T3 作為預測目標，而非 T2，主要有兩個原因：
>
> 第一，這樣才能使用 Δ 特徵。Δ 是 T2 減 T1 的變化量，如果目標也是 T2，就會造成資料洩漏。
>
> 第二，這樣才是真正的「預測」——用過去的資料預測未來的健康狀態，給醫療人員約兩年的介入時間窗口。
