## 其他討論事項

### 5. 加入 Decision Tree (DT) 模型

- **教授建議**：加入 DT 決策樹模型
- **與 RF 的關係**：Random Forest = 多棵 Decision Tree 的集成
- **好處**：
  - **可解釋性高**：可以畫出決策樹，直觀理解分類邏輯
  - **基準比較**：DT 是 RF 的基礎，可看集成帶來多少提升
  - **文獻常見**：很多論文會同時報告 DT 和 RF 結果

### 6. MTL（Multi-Task Learning）是重點實驗

- **地位**：MTL 是本研究的重點之一，需要放入正式實驗
- **目前狀態**：已有初步嘗試（03_ModelBuilding），但未深入分析
- **待做**：
  - 完整比較 STL vs MTL 的效能差異
  - 分析 MTL 在三高預測上的優勢

### 7. PySR 樹深度實驗

- **問題**：目前 PySR 使用 default 深度設定
- **建議**：嘗試不同深度，看對公式複雜度和效能的影響
- **待做**：

  - 測試 depth = 3, 5, 7, 10 等不同設定
  - 比較公式可解釋性 vs 效能的 trade-off

### 8. 健檢次數與預測準確度實驗

- **參考**：Taiwan MJ 論文發現「訪問次數越多，預測越準」
- **Taiwan MJ 做法**：
  - 把多次健檢特徵串接（T1 + T2 + ... + TN）
  - 比較 1 次、2-3 次、4+ 次健檢的預測效能
  - 結果：4+ 次最準
- **我們可以做**：
  - 目前固定 3 次（T1, T2 → 預測 T3）
  - 可擴展為：T1 → T2、T1+T2 → T3、T1+T2+T3 → T4
  - 驗證「次數越多越準」的假設是否成立
